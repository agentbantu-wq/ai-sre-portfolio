# AI/SRE Project: PowerGrid Sentinel

## 1. Project Title
**PowerGrid Sentinel** — Real-time AI Workload Power Consumption Predictor and Alerting System

## 2. Problem Statement
With AI infrastructure spending projected to hit $400-450B in 2026, hyperscalers like Microsoft (25GW capacity) and Amazon ($200B capex) face massive power grid strain from GPU-heavy workloads[1][2]. SRE teams lack predictive tools to forecast power draw, prevent grid overloads, and trigger cost-saving autoscaling or throttling before outages or skyrocketing energy bills hit.

## 5. Minimal Code Skeleton
### `predictor/main.py` (FastAPI + XGBoost)
```python
from fastapi import FastAPI
import xgboost as xgb
import numpy as np
from pydantic import BaseModel
import joblib  # For model loading

app = FastAPI()
model = joblib.load("power_predictor.model")  # Train & save on Day 3

class PowerInput(BaseModel):
    gpu_util: float
    num_gpus: int
    req_rate: float  # Tokens/sec

@app.post("/predict")
def predict_power(data: PowerInput):
    features = np.array([[data.gpu_util, data.num_gpus, data.req_rate]])
    pred_kw = model.predict(features)[0]  # Predicted kW
    alert = pred_kw > 500  # Threshold: 500kW/cluster
    return {"predicted_kw": pred_kw, "alert": alert, "action": "scale_down" if alert else "ok"}
```

## 7. Failure Mode Analysis
| Failure Mode | Impact | Mitigation |
|--------------|--------|------------|
| ML Model Drift (new GPUs) | False alerts | Weekly retrain cronjob + canary validation |
| Prometheus Scrape Fail | Blind scaling | Fallback to avg historical power + circuit breaker |
| K8s Autoscaler Lag | Over-provision | Max replicas cap + predictive lead time (predict 5m ahead) |
| High Cardinality Metrics | OOM in Prom | Relabel + downsampling (e.g., per-node only) |
| Graceful Degradation | N/A | Static thresholds if ML offline |

## 8. Observability Design
- **Metrics**: `power_predicted_kw`, `power_actual_kw`, `scale_events_total`, `alert_fired` (Prometheus).
- **Logs**: Structured JSON via Fluentd → Loki (e.g., `{"level":"warn","power_exceeded":true}`).
- **Traces**: OpenTelemetry on FastAPI (Jaeger) for prediction latency.
- **Dashboards**: Grafana with panels for forecast vs actual, anomaly heatmaps.
- **Alerts**:
  ```
  - alert: PowerOverload
    expr: power_predicted_kw > 450
    for: 2m
    annotations: {summary: "Predicted overload in 5m"}
  ```
